


Experiment A:
- launch_experiment_a.py: The SLURM job submission script to sweep over the baseline and CPPO configurations.
- prm_mcts_oracle.py: An auxiliary module to simulate or extract the step-level Process Reward Model confidences using Upper Confidence Bounds (UCB) for optimistic exploration.
- grpo_cppo_train_exp_a.py: The main training script containing the custom loss and logging hooks to extract $\text{Var}(\rho_t)$ across the num_generations dimension.

Experiment B:
- launch_experiment_b.py: This script sweeps through the three exploration modes, launching parallel jobs to isolate their effects on AIME validation scores and reasoning chain integrity.
- langevin_exploration.py: Epistemic baseline utility. To implement the continuous exploration baseline, we inject Langevin noise directly into the advantage estimates. This forces the policy to explore by artificially inflating or deflating the gradient signal based on a simulated uncertainty-driven step size.
- grpo_cppo_train_exp_b.py: Main training script. This script integrates both the Langevin baseline and the PRM-guided bounds (which rely on the prm_mcts_oracle.py we built for Experiment A). We apply Upper Confidence Bounds (UCB) for optimistic exploration to extract the true step-level structural uncertainty.

Experiment C:
- launch_experiment_c.py
- grpo_cppo_train_exp_c.py: This file introduces cppo_gamma to the configuration and overrides the standard torch.cumsum with a vectorized exponentially weighted moving average.

Experiment D:
- launch_experiment_d.py
- epistemic_prm_oracles.py: This file implements the two distinct formalizations of step-level uncertainty. The UCT baseline utilizes optimistic exploration via upper confidence bounds, while the Epinet models an explicit variance readout over the PRM's hidden states.
- grpo_cppo_train_exp_d.py: This script integrates the epistemic oracles into the main loss loop, dynamically adjusting the trust region ($\epsilon_t$) based on the specified formalization.
